# 133 — Client Onboarding Flow

## Objetivo

Definir o fluxo de onboarding de um novo cliente dentro do SEO20.

## Etapas

1. Criar cliente.
2. Cadastrar site principal.
3. Definir segmento.
4. Definir região de atuação quando aplicável.
5. Conectar integrações disponíveis.
6. Configurar concorrentes.
7. Executar primeira auditoria.
8. Gerar primeiro relatório.
9. Criar plano de ação.

## Dados iniciais

- Nome do cliente.
- Site.
- Segmento.
- Região.
- Concorrentes.
- Objetivos.
- Observações comerciais.

## Checklist

- Site validado.
- Cliente vinculado à organização.
- Usuários internos com acesso.
- Integrações autorizadas.
- Auditoria inicial concluída.
- Relatório revisado.

## Regras

- Cliente deve pertencer a uma organização.
- Site deve ter URL válida.
- Integrações são opcionais no MVP.
- Onboarding pode ficar incompleto sem bloquear auditoria técnica.

## Evolução futura

- Wizard guiado.
- Importação em lote.
- Questionário do cliente.
- Onboarding white-label.
- Convite automático ao portal.

# 91 — Dashboard UX Guidelines

## Objetivo

Definir diretrizes de experiência para dashboards do SEO20, garantindo leitura rápida, priorização clara e navegação eficiente.

## Princípios

- Mostrar primeiro o que exige ação.
- Separar visão executiva de visão técnica.
- Evitar excesso de métricas na tela inicial.
- Usar scores com explicação e contexto.
- Priorizar comparação histórica e tendência.
- Permitir drill-down para evidências.

## Estrutura recomendada

### Visão geral

- Score geral.
- Principais alertas.
- Oportunidades prioritárias.
- Evolução recente.
- Status de auditorias.

### Áreas

- SEO Técnico.
- SEO Local.
- GEO.
- CRO.
- Ads.
- Backlinks.
- Conteúdo.

### Ações

- Recomendações abertas.
- Recomendações concluídas.
- Próximos passos.
- Impacto estimado.

## Estados da interface

Toda tela deve prever:

- Loading.
- Empty state.
- Erro.
- Dados parciais.
- Integração desconectada.
- Permissão insuficiente.

## Hierarquia visual

Prioridade:

1. Status crítico.
2. Score geral.
3. Ações recomendadas.
4. Tendências.
5. Detalhes técnicos.

## Dashboard para agência

Deve mostrar:

- Carteira de clientes.
- Clientes em risco.
- Melhores oportunidades.
- Auditorias pendentes.
- Relatórios gerados.
- Consumo do plano.

## Dashboard do cliente

Deve mostrar:

- Resultado simplificado.
- Evolução.
- O que foi feito.
- O que será feito.
- Relatórios compartilhados.

## Regras

- Não mostrar dado sem fonte quando for métrica sensível.
- Não misturar clientes na mesma visualização sem indicação clara.
- Sempre indicar data da última atualização.
- Gráficos devem ter legenda e unidade.

## Evolução futura

- Dashboards customizáveis.
- Widgets por papel.
- Alertas inteligentes.
- Comparação entre clientes.
- Modo apresentação para reuniões.

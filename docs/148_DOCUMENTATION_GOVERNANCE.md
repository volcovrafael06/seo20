# 148 — Documentation Governance

## Objetivo

Definir como a documentação do SEO20 deve ser mantida, revisada e evoluída junto com o produto.

## Princípios

- Documento técnico deve refletir decisão atual.
- Mudanças estruturais exigem atualização documental.
- Documentos obsoletos devem ser marcados.
- Decisões importantes devem virar ADR.
- Documentação deve ser objetiva e rastreável.

## Tipos de documento

- Arquitetura.
- Produto.
- Banco.
- API.
- Operação.
- Segurança.
- IA.
- UX.
- Roadmap.

## Status

- draft.
- active.
- deprecated.
- archived.

## Revisão

Revisar documentação quando houver:

- Nova feature estrutural.
- Mudança em banco.
- Mudança em API.
- Mudança em permissão.
- Mudança em integração.
- Mudança em workflow.

## Evolução futura

- Índice automático.
- Donos por seção.
- Revisão trimestral.
- Checklist de PR para docs.

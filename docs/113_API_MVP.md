# 113 — API MVP

## Objetivo

Registrar os grupos de API necessários para o MVP.

## Grupos

- Health.
- Organizations.
- Members.
- Clients.
- Websites.
- Audits.
- Findings.
- Recommendations.
- Reports.

## Regras

- Todas as rotas devem validar organização.
- Listagens devem ter paginação.
- Tarefas longas devem usar fila.
- Permissões devem ser verificadas no backend.

## Resultado esperado

A API deve permitir cadastrar cliente, cadastrar site, executar auditoria, listar recomendações e gerar relatório.

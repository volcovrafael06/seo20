# 138 — AI Governance

## Objetivo

Definir regras de governança para uso de IA no SEO20.

## Princípios

- IA deve ter escopo definido.
- Saídas devem ser revisáveis.
- Decisões críticas exigem aprovação humana.
- Custos devem ser monitorados.
- Dados devem respeitar permissões.

## Áreas cobertas

- Agentes.
- Prompts.
- RAG.
- Model Router.
- Relatórios.
- Recomendações.
- Automações assistidas.

## Regras

- Não inventar métricas.
- Separar fato de hipótese.
- Registrar modelo usado.
- Registrar custo estimado.
- Validar formato de saída.
- Exigir aprovação para ações externas.

## Qualidade

Monitorar:

- Taxa de saída válida.
- Correções humanas.
- Custo por tarefa.
- Tempo de resposta.
- Feedback do usuário.

## Evolução futura

- Avaliações automáticas.
- Painel de qualidade.
- Comparação entre modelos.
- Políticas por plano.

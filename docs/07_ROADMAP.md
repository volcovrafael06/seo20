# 07 — Roadmap

## Visão geral

O SEO20 será desenvolvido por fases para gerar valor rapidamente e permitir evolução segura.

## Fase 0 — Fundação

Objetivo: organizar documentação, arquitetura e base técnica.

Entregas:

- Repositório GitHub.
- Documentação inicial.
- Definição de stack.
- Modelo de dados.
- Roadmap.
- Backlog inicial.

## Fase 1 — MVP de auditoria

Objetivo: permitir cadastrar cliente e gerar uma auditoria inicial.

Entregas:

- Auth Supabase.
- Cadastro de organizações.
- Cadastro de clientes.
- Criação de auditorias.
- SEO técnico básico.
- Captura de páginas principais.
- Scores iniciais.
- Relatório PDF simples.

## Fase 2 — SEO e concorrência

Objetivo: tornar a auditoria comercialmente forte.

Entregas:

- Keyword Engine.
- SERP Engine.
- Descoberta de concorrentes.
- SEO local.
- Google Maps.
- Google Business Profile.
- Benchmark.
- PDF executivo completo.

## Fase 3 — Backlinks e autoridade

Objetivo: incluir análise de autoridade.

Entregas:

- Backlink Engine.
- Link Gap.
- Backlinks tóxicos.
- Oportunidades seguras.
- Digital PR.
- Monitoramento semanal.

## Fase 4 — Tráfego pago

Objetivo: conectar dados de Ads.

Entregas:

- Google Ads API.
- Meta Marketing API.
- Auditoria de campanhas.
- Análise de pixel/eventos.
- Verba desperdiçada.
- Sugestões de campanhas.

## Fase 5 — Dashboard avançado

Objetivo: transformar o sistema em plataforma de acompanhamento.

Entregas:

- Dashboard executivo.
- Histórico de auditorias.
- Evolução de scores.
- Alertas.
- Tarefas.
- Permissões por função.

## Fase 6 — IA e automações

Objetivo: gerar planos e ações assistidas.

Entregas:

- Agentes especializados.
- Prompts versionados.
- RAG com base de conhecimento.
- Geração de conteúdo.
- Sugestão de páginas.
- Geração de campanhas.
- Geração de tarefas.

## Fase 7 — Correção automática

Objetivo: executar mudanças com aprovação humana.

Entregas:

- Integração WordPress.
- Integração GitHub.
- Abertura de PRs.
- Correções de SEO.
- Criação de landing pages.
- Campanhas Google Ads.
- Campanhas Meta Ads.
- Logs e rollback.

## Fase 8 — SaaS white-label

Objetivo: vender para outras agências.

Entregas:

- Multiempresa completo.
- Billing.
- Planos.
- Limites de uso.
- Domínio personalizado.
- Marca branca.
- API pública.

## Prioridade inicial

Começar pelo MVP com:

1. Supabase.
2. Next.js.
3. FastAPI.
4. Cadastro de cliente.
5. Auditoria técnica simples.
6. Relatório PDF.
7. Dashboard básico.

Depois evoluir para concorrentes, backlinks e Ads.

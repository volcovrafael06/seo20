# 19 — Dashboard e UX

## Objetivo

Definir a experiência de uso da plataforma SEO20. O dashboard deve ajudar a agência a enxergar clientes, riscos, oportunidades, auditorias e próximos passos.

## Princípios de UX

1. Mostrar prioridade, não apenas dados.
2. Traduzir informações técnicas em impacto comercial.
3. Reduzir o tempo entre análise e proposta.
4. Deixar claro o status de cada auditoria.
5. Permitir acesso rápido ao PDF executivo.
6. Funcionar para analistas e vendedores.

## Dashboard principal

Widgets principais:

- Total de clientes ativos.
- Auditorias em andamento.
- Relatórios gerados no mês.
- Score médio dos clientes.
- Oportunidades de alta prioridade.
- Clientes com maior risco.
- Integrações pendentes.

## Tela de cliente

Blocos:

- Identificação.
- Site e canais.
- Score atual.
- Histórico de auditorias.
- Recomendações abertas.
- Relatórios.
- Integrações.

## Tela de auditoria

A tela de auditoria deve ser o centro operacional.

Seções:

- Header com nome do cliente, status e botão gerar PDF.
- Progresso por módulo.
- Scores por área.
- Recomendações priorizadas.
- Concorrentes.
- Palavras-chave.
- Backlinks.
- Ads.
- Logs resumidos.

## Timeline de auditoria

Exemplo de eventos:

- Auditoria criada.
- Crawler iniciado.
- Keywords coletadas.
- Concorrentes identificados.
- SEO técnico concluído.
- Backlinks analisados.
- Recomendações geradas.
- PDF gerado.

## Centro de oportunidades

Visão agregada de recomendações de todos os clientes.

Filtros:

- Cliente.
- Módulo.
- Prioridade.
- Impacto.
- Esforço.
- Status.

Status:

- Aberta.
- Aprovada.
- Em execução.
- Concluída.
- Ignorada.

## Experiência para vendas

Criar uma visão comercial com:

- Diagnóstico rápido.
- Principais dores.
- Estimativa de impacto.
- Botão gerar proposta.
- Botão baixar PDF.

## Experiência para analista

Criar visão técnica com:

- Dados brutos.
- Logs.
- Evidências.
- Exportação CSV.
- Reprocessamento de módulos.

## Experiência para cliente

Acesso limitado:

- Ver relatórios.
- Ver evolução.
- Ver tarefas aprovadas.
- Baixar PDFs.

## Estados de interface

### Empty state

Sempre orientar o próximo passo.

Exemplo: "Você ainda não cadastrou nenhum cliente. Cadastre o primeiro cliente para gerar uma auditoria."

### Loading

Mostrar progresso e mensagem clara.

### Erro

Mostrar erro amigável e opção de tentar novamente.

### Sucesso

Confirmar ação e sugerir próximo passo.

## Métricas visuais

Usar cores com consistência:

- Verde: bom/concluído.
- Amarelo: atenção.
- Vermelho: crítico.
- Azul: informação.
- Roxo: IA/automação.

## Navegação

Menu principal:

- Dashboard.

- Auditorias.
- Oportunidades.
- Relatórios.
- Integrações.
- Configurações.

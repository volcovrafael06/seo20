# 116 — Recommendation Engine MVP

## Objetivo

Definir como o SEO20 transforma achados de auditoria em recomendações acionáveis no MVP.

## Entrada

- Achados da auditoria.
- Categoria do problema.
- Severidade.
- Quantidade de URLs afetadas.
- Evidência disponível.

## Saída

Cada recomendação deve conter:

- Título.
- Descrição.
- Prioridade.
- Impacto.
- Esforço.
- Status.
- Evidência.
- Próximo passo.

## Prioridade

Critérios:

- Severidade técnica.
- Volume de páginas afetadas.
- Impacto em indexação.
- Impacto em conversão.
- Facilidade de correção.

## Status

- open.
- planned.
- in_progress.
- done.
- ignored.

## Regras

- Toda recomendação deve nascer de evidência.
- Não duplicar recomendações idênticas.
- Agrupar problemas recorrentes.
- Permitir atualização manual de status.

## Evolução futura

- Priorização por IA.
- Estimativa de impacto.
- Atribuição de responsável.
- Aprovação de automação.

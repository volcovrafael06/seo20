# 62 — Sales Engine

## Objetivo

O Sales Engine transforma diagnósticos técnicos do SEO20 em argumentos comerciais claros para venda, retenção e expansão de contratos de marketing.

## Problema resolvido

Auditorias técnicas frequentemente mostram muitos dados, mas não explicam com clareza o impacto comercial. O Sales Engine converte problemas em oportunidades de negócio.

## Entradas

- Scores da auditoria.
- Recomendações priorizadas.
- Dados de tráfego.
- Dados de conversão.
- Benchmarks concorrenciais.
- Dados de Ads.
- Dados de CRO.
- Histórico do cliente.
- Segmento de mercado.

## Responsabilidades

- Gerar narrativa comercial.
- Traduzir problemas técnicos em impacto financeiro.
- Identificar oportunidades de upsell.
- Preparar argumentos para reunião.
- Criar resumo executivo para decisores.
- Sugerir próximos serviços.
- Apoiar propostas comerciais.

## Tipos de saída

- Resumo para reunião.
- Diagnóstico comercial.
- Matriz problema/impacto/solução.
- Oportunidades de upsell.
- Objeções prováveis.
- Script de apresentação.
- Plano de 30/60/90 dias.

## Exemplo de tradução

Problema técnico:

- Meta titles duplicados em páginas de serviço.

Tradução comercial:

- As páginas competem entre si e reduzem a chance de capturar buscas qualificadas. Corrigir isso pode melhorar a clareza para o Google e aumentar a entrada de leads orgânicos.

## Matriz de priorização comercial

Cada oportunidade deve receber:

- Impacto em receita.
- Urgência.
- Facilidade de execução.
- Dependência técnica.
- Risco de não executar.
- Potencial de upsell.

## Integração com Proposal Engine

O Sales Engine alimenta o Proposal Engine com pacotes sugeridos, escopo, justificativa e ordem de prioridade.

## Regras

- Não inventar números financeiros.
- Quando estimar, marcar como estimativa.
- Usar linguagem compreensível para decisores.
- Separar achado técnico de recomendação comercial.
- Evitar promessas absolutas.
